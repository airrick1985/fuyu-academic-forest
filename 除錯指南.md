## 🎯 常見錯誤和解決方案

### ❌ 錯誤 1: "Cannot find module 'electron'"

**症狀：** 打包失敗，提示找不到 electron 模塊

**解決方案：**
```bash
# 清除 node_modules 並重新安裝
rm -rf node_modules
npm install
npm list electron
```

---

### ❌ 錯誤 2: "Assets not found 404"

**症狀：** EXE 運行後，CSS/JS/圖片無法加載

**診斷：**
```bash
# 1. 檢查 assets-manifest.json 是否存在
cat assets-manifest.json | head -20

# 2. 檢查打包後的文件結構
ls -R dist/win-unpacked/resources/app/ | grep -E "(css|js|assets)"

# 3. 檢查 Service Worker 日誌
# 在 EXE 中按 F12 → 控制台 → 搜索 "[SW]"
```

**解決方案：**
```bash
# 確保 package.json 的 files 配置包含所有資源
# 檢查當前配置
npm list | grep -A 50 '"files"'

# 如果不完整，重新生成清單
npm run generate-manifest
npm run build
```

---

### ❌ 錯誤 3: "白屏 (Blank Screen)"

**症狀：** EXE 啟動但窗口顯示白屏

**診斷步驟：**

1️⃣ **檢查服務器是否啟動**
```javascript
// 在 DevTools 控制台執行
fetch('http://localhost:3456/')
  .then(r => console.log('✓ 服務器正常'))
  .catch(e => console.log('✗ 服務器連接失敗:', e))
```

2️⃣ **檢查 index.html 是否被加載**
```javascript
// 在控制台執行
console.log('當前 URL:', window.location.href)
console.log('文檔內容:', document.documentElement.innerHTML.length, '字節')
```

3️⃣ **檢查是否有資源 404**
```javascript
// 查看 Network 標籤，找到失敗的請求
// 記住文件名和路徑
```

**解決方案：**

如果是路徑問題，檢查 `electron/main.js`：
```javascript
// 應該看到
const serverUrl = startLocalServer();  // http://localhost:3456
mainWindow.loadURL(serverUrl);
```

如果是 Service Worker 問題，檢查 `sw.js` 的路徑轉換：
```javascript
// 應該正確檢測 Electron 環境
const IS_ELECTRON = typeof window !== 'undefined' && window.isElectron === true;
```

### ❌ 錯誤 4: "SPA 路由無法正常工作"

**症狀：** 點擊頁面導航無反應

**檢查步驟：**
```javascript
// 在控制台檢查路由器
console.log('路由器狀態:', window.router || '未初始化')

// 檢查 Service Worker 狀態
navigator.serviceWorker.getRegistrations()
  .then(regs => {
    console.log('SW 數量:', regs.length)
    regs.forEach(reg => console.log('SW 作用域:', reg.scope))
  })
```

**解決方案：**

檢查 `js/spa-router.js` 中的路由配置，確保使用相對路徑：
```javascript
// ✓ 正確 - 相對路徑
const routes = {
  '/': 'index.html',
  '/about': 'about.html'
}

// ✗ 錯誤 - 絕對路徑（在 Electron 中會失效）
const routes = {
  '/': '/index.html',
}
```

### ❌ 錯誤 5: "EXE 無法在其他電腦上運行"

**症狀：** 在一台電腦上能運行，在另一台電腦上不行

**可能原因：**
- [ ] 缺少 Visual C++ 運行庫
- [ ] 依賴的 DLL 文件不完整
- [ ] 防火牆阻止 localhost:3456

**解決方案：**
```bash
# 1. 檢查 EXE 是否包含所有依賴
# 使用 7-Zip 或 WinRAR 打開 EXE，檢查結構
# 應該看到 resources/app 包含所有文件

# 2. 安裝 Visual C++ 運行庫
# 下載: https://support.microsoft.com/en-us/help/2977003

# 3. 測試端口可用性
netstat -ano | findstr 3456
```

## 🔍 深度除錯技巧

### 技巧 1: 查看詳細日誌
```bash
# 執行 EXE 時輸出日誌到文件
npm run electron-build-win -- --verbose > build.log 2>&1

# 查看日誌
cat build.log
```

### 技巧 2: 檢查打包配置
```bash
# 驗證 electron-builder 配置
npx electron-builder --help

# 查看當前配置
npx electron-builder --publish never -c --dry-run
```

### 技巧 3: 測試開發版本
```bash
# 這會保留符號和調試信息，便於除錯
npm run electron-build-win -- --dir
```

### 技巧 4: 監控進程
```bash
# 檢查 EXE 運行時的進程
tasklist | findstr "富宇學森"

# 查看進程詳細信息
Get-Process | Where-Object { $_.ProcessName -like "*electron*" }
```
