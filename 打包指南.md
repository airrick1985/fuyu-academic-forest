## 📦 打包步驟（詳細版）

### 步驟 1: 清理舊的構建文件
```bash
# 刪除之前的打包結果
rmdir /s dist
rmdir /s out
del *.exe
```

### 步驟 2: 確保依賴安裝完整
```bash
npm install
npm list electron electron-builder
```

應該看到：
```
+-- electron-builder@26.8.1

```

```bash
npm run build
```

這會自動執行 `generate-manifest.js` 並生成 `assets-manifest.json`

```bash
npm run electron-build-win
```

### 步驟 5: 檢查輸出
打包完成後，檢查：
```
📁 dist/
```

## 🐛 常見問題和除錯

### 問題 1: 資源 404（找不到 CSS、JS、圖片）

**原因：** assets-manifest.json 未生成或不完整

**解決方案：**
```bash
# 重新生成清單
npm run generate-manifest

# 檢查生成結果
cat assets-manifest.json
```

應該看到：
```json
{
  "version": "1.0.3",
  "assets": {
    "css/variables.css": { "hash": "abc123..." },
    ...
  }
}
```

### 問題 2: 白屏或無法連接服務器

**原因：** Service Worker 路徑問題或端口被占用

**除錯步驟：**
```bash
# 1. 檢查 Service Worker 是否註冊
# 在 EXE 中按 F12，進入 Console，執行：
navigator.serviceWorker.getRegistrations()
  .then(regs => console.log('SW註冊:', regs))

# 2. 檢查網絡連接
fetch('http://localhost:3456/')
  .then(r => r.text())
  .then(t => console.log('連接成功', t.substring(0, 100)))
  .catch(e => console.error('連接失敗:', e))

# 3. 檢查本地服務器日誌
# 如果服務器沒有啟動，會有錯誤信息在控制台
```

### 問題 3: EXE 創建後無法執行

**原因：** electron-builder 配置錯誤

**解決方案：**
```bash
# 檢查配置
npm run electron-build-win -- --publish never -d

# 詳細輸出
npm run electron-build-win -- --publish never --verbose
```

### 問題 4: 關於菜單中文亂碼

**原因：** 文件編碼問題

**解決方案：** main.js 已用 UTF-8 寫入，應該正常

### 問題 5: 多次打包導致更新問題

**原因：** 版本號未更新

**解決方案：**
```json
// package.json 中更新版本
{
  "version": "1.0.4"  // 改這個
}
```

同時更新 index.html：
```html
<meta name="app-version" content="1.0.4">
```

## 📊 完整打包流程（一鍵執行）

創建一個 `build.bat` 文件：

```batch
@echo off
echo ===== 清理舊文件 =====
rmdir /s /q dist 2>nul
rmdir /s /q out 2>nul

echo ===== 安裝依賴 =====
call npm install

echo ===== 生成清單 =====
call npm run build

echo ===== 構建 EXE =====
call npm run electron-build-win

echo.
echo ===== 打包完成 =====
echo 輸出位置: dist\
pause
```

然後雙擊 `build.bat` 即可
