## 📝 完整步驟發布

### 方式 1️⃣：分行執行（最安全、最清晰）

```powershell

npm run build

# 2. 添加所有改動
git add .

# 3. 提交
git commit -m "更新資源及清單"

# 4. 推送到 GitHub（自動部署）
git push origin main
```

### 方式 2️⃣：一行指令（PowerShell）

```powershell
npm run build; git add .; git commit -m "更新資源及清單"; git push origin main
```

### 方式 3️⃣：一行指令（Bash/Git Bash）

```bash
npm run build && git add . && git commit -m "更新資源及清單" && git push origin main
```

## 🎯 常見場景

### 場景 1：更新單個資源文件

```powershell
# 例如：更新 A1.webp
Copy-Item assets/images/layout/new-A1.webp assets/images/layout/A1.webp -Force

# 生成清單並發布
npm run build; git add .; git commit -m "更新資源：修改 A1.webp"; git push origin main
```

### 場景 2：更新多個資源文件

```powershell
# 1. 複製/編輯多個資源文件
Copy-Item path/to/new-resource-1.webp assets/images/layout/A1.webp -Force
Copy-Item path/to/new-resource-2.webp assets/images/layout/A2.webp -Force

# 2. 發布
npm run build; git add .; git commit -m "更新資源：修改多個文件"; git push origin main
```

### 場景 3：更新代碼 + 資源

```powershell
# 1. 編輯代碼和資源文件
# ... 修改 js/ 或 css/ 文件 ...
# ... 更新 assets/ 文件 ...

# 2. 生成清單
npm run build

# 3. 提交詳細信息
git add .
git commit -m "feat: 更新功能和資源

- 更新 js/topbar.js 功能
- 更新資源文件
- 清單已自動生成
"

git push origin main
```

### 場景 4：只提交代碼（不涉及資源）

```powershell
# 如果只修改 JS/CSS/HTML，不涉及資源
git add .
git commit -m "fix: 修複 Service Worker 邏輯"
git push origin main
```

## 🚨 常見錯誤

### ❌ 錯誤 1：忘記執行 `npm run build`

**解決**：
```powershell
npm run build
git add assets-manifest.json
git commit -m "更新清單"
git push origin main
```

### ❌ 錯誤 2：PowerShell 中使用 `&&`

**解決**：
```powershell
# ✅ 改用 ;（PowerShell 語法）
npm run build; git add .; git commit -m "..."; git push origin main
```

### ❌ 錯誤 3：提交前忘記 `git add`

**解決**：
```powershell
git add .
git status  # 確認有待提交的改動
git commit -m "..."
```
